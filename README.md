## Production Setup

Copy `.env.example` to `.env.local` and set:

- `AUTH_SECRET` to a long random secret.
- `PI_SERVER_API_KEY` from the Pi Developer Portal.
- `NEXT_PUBLIC_PI_SANDBOX=true` while testing in Pi sandbox.
- `NEXT_PUBLIC_PI_BASIC_AMOUNT`, `NEXT_PUBLIC_PI_PREMIUM_AMOUNT`, and `NEXT_PUBLIC_PI_ELITE_AMOUNT` to the Pi amounts you want to charge.
- `DATABASE_URL` before moving away from the in-memory demo store.
- `DATABASE_SSL=false` only for local non-SSL Postgres. Hosted production databases should normally keep SSL enabled.

Pi payments are started in the subscription page with the Pi Browser SDK. The app then calls:

- `/api/pi/payments/approve` when Pi reports the payment is ready for server approval.
- `/api/pi/payments/complete` when Pi reports the transaction ID is ready for completion.

Subscriptions should only unlock after the complete endpoint verifies the Pi payment and updates the authenticated account.

## Deploy

This app should be deployed to a host that can run a Next.js server because it
uses API routes for authentication, subscriptions, and Pi Network payments.

### Railway

Railway is the recommended first deployment target. The repo includes:

- `railway.json` with build, start, healthcheck, and restart settings.
- `next.config.mjs` with `output: "standalone"` for the production server.
- `npm run start:railway` to run `.next/standalone/server.js`.
- `Dockerfile` and `.dockerignore` for a Docker-based deployment if needed.

Quick start:

```bash
npm install
npm run build
npm run start:railway
```

In Railway:

1. Create a new project from the repository.
2. Add Postgres or connect an external Postgres database.
3. Set the production environment variables from `.env.example`.
4. Add `unitydating.net` and `www.unitydating.net` under service networking.
5. Copy Railway's DNS records into Unstoppable Domains.

## Database

The app uses Postgres when `DATABASE_URL` is set. It automatically creates a
minimal `unity_accounts` table for account/session and subscription state.
Without `DATABASE_URL`, local development uses an in-memory fallback that resets
when the server restarts. Do not launch with the fallback store.
